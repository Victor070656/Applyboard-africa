PROJECT OVERVIEW
This document defines the full developer-ready build specification for the ApplyBoard Africa Ltd website and digital platform. 
The system supports public marketing pages, student/client portals, agent dashboards, and an admin operations backend. 
The platform is built using PHP and is designed for scalability, security, and operational clarity.

1. GOALS AND SCOPE
Business Goals:
- Provide a modern education and travel consulting website.
- Capture qualified leads for study abroad, visa, and travel services.
- Enable a digital platform for application tracking and future integrations.
- Support verified agents, referral tracking, performance ratings, and commissions.

Project Phases:
Phase 1: Marketing website + inquiry capture.
Phase 2: Student, Agent, and Admin portals.
Phase 3: Automation, reporting, commissions, and integrations.

Key Rule:
Once a client is linked to an agent, the relationship is permanently locked unless overridden by admin.

7. AGENT WORKFLOW

1. Agent registers.
2. Submits verification documents.
3. Admin reviews and verifies.
4. System generates agent ID and referral link.
5. Agent can submit or register clients.
6. Client remains permanently linked to agent.
7. Agent performance is rated (activity, quality, outcomes).

8. REFERRAL TRACKING LOGIC

- Referral link: https://domain.com/register?ref=AGENTSLUG
- Manual agent code allowed.
- Middleware captures referral.
- Referral applied on inquiry or registration.
- Permanent agent-client linkage enforced.

9. CASE TYPES AND STAGES

Study Abroad:
Assessment → Options → Application → Submission → Offer → Visa → Travel → Closed

Visa:
Assessment → Documents → Submission → Decision → Closed

Travel:
Requirements → Booking → Completed → Closed
